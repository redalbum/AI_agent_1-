# Telegram Bot — Бизнес-ассистент 1С

Универсальный Telegram-бот для работы с любой конфигурацией 1С.  
При первом запуске сканирует метаданные базы и адаптируется под конкретную конфигурацию.

## Архитектура

```
Telegram Bot (aiogram 3.x)
    ↓
Python Backend (async)
    ↓
1С через COM (win32com)
    ↓
OpenRouter / LLM (анализ и рекомендации)
```

## Структура файлов

```
automation/telegram_bot/
├── bot.py               # Главный файл бота, точка входа
├── config.py            # Загрузка конфигурации из .env
├── prompts.py           # Системные промпты для LLM
├── llm_client.py        # Клиент для OpenRouter / OpenAI-совместимых API
├── onec_connector.py    # Обёртка COM-подключения к 1С
├── metadata_scanner.py  # Сканер метаданных 1С с кешированием
├── handlers/
│   ├── analytics.py     # Аналитика продаж (топ, аутсайдеры, динамика)
│   ├── inventory.py     # Склад и закупки (остатки, дефицит, залежалые)
│   ├── finance.py       # Финансы (дебиторка, кредиторка, касса)
│   └── free_question.py # Свободный вопрос к базе через LLM
└── README.md
```

## Требования

- **Python 3.11+** (рекомендуется)
- **Windows** (для COM-подключения к 1С)
- **1С:Предприятие 8.x** с установленной платформой
- Токен Telegram-бота (получить через [@BotFather](https://t.me/BotFather))
- API-ключ LLM провайдера (OpenRouter или совместимый)

## Установка

### 1. Установите зависимости

```bash
# Из каталога automation/
pip install -r requirements.txt
pip install -r requirements-com.txt  # для COM на Windows
```

### 2. Настройте .env

Скопируйте `.env.example` в `.env` в каталоге `automation/telegram_bot/` и заполните:

```ini
# Токен бота от @BotFather
TELEGRAM_BOT_TOKEN=1234567890:AABBccDDeeFFggHHiiJJkkLL

# Список разрешённых Telegram user_id (пусто = доступ для всех)
TELEGRAM_ALLOWED_USERS=123456789

# Путь к папке с базой 1С
ONEC_BASE_PATH=C:\Bases\Retail
ONEC_USER=
ONEC_PASSWORD=

# API-ключ OpenRouter (openrouter.ai)
OPENROUTER_API_KEY=sk-or-v1-...

# Модель ИИ
OPENROUTER_MODEL=anthropic/claude-sonnet-4

# Кеш метаданных (часов)
METADATA_CACHE_HOURS=24
```

### 3. Получите токен бота

1. Найдите [@BotFather](https://t.me/BotFather) в Telegram
2. Отправьте `/newbot`
3. Следуйте инструкциям, получите токен вида `1234567890:AABBcc...`
4. Вставьте токен в `.env` как `TELEGRAM_BOT_TOKEN`

### 4. Узнайте ваш Telegram user_id

Отправьте любое сообщение боту [@userinfobot](https://t.me/userinfobot) — он покажет ваш ID.  
Добавьте его в `TELEGRAM_ALLOWED_USERS` для ограничения доступа.

## Запуск

```bash

python -m automation.telegram_bot.bot
```

## Логика работы

1. **Первый запуск** → `/start` → предложение сканировать → кеширование в `metadata_cache.json`
2. **Обычный запуск** → загрузка кеша (если не устарел) → главное меню
3. **Аналитика** → выбор отчёта → выбор периода → запрос к 1С → LLM-анализ → ответ
4. **Свободный вопрос** → LLM генерирует запрос 1С → выполнение → LLM форматирует ответ

## Кеширование метаданных

- Кеш хранится в файле `metadata_cache.json` (путь настраивается через `METADATA_CACHE_FILE`)
- Время жизни — `METADATA_CACHE_HOURS` часов (по умолчанию 24)
- Для принудительного обновления: `/scan` или ⚙️ Настройки → Пересканировать

## Ограничение доступа

Задайте `TELEGRAM_ALLOWED_USERS` со списком разрешённых user_id через запятую.  
Если переменная пустая — доступ открыт для всех.

## Обработка ошибок

- Если 1С недоступна — бот сообщает об ошибке и предлагает попробовать позже
- Если LLM недоступен — данные показываются без анализа/рекомендаций
- Все ошибки логируются в консоль
