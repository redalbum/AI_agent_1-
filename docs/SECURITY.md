# Безопасность

Агент выполняет действия в рамках прав текущего пользователя 1С. Дополнительно предусмотрены ограничения на запись.

## Режим Запрос1С (только чтение)

Тип диалога **ИИА_ТипДиалога.Запрос1С** запрещает все действия изменения данных:

- **Запрещённые действия:** CreateReference, SetField, Write, Delete
- **Разрешённые:** GetMetadata, GetObjectFields, CheckObjectExists, RunQuery, ShowInfo, SelectObject (чтение)

Проверка выполняется в **ИИА_DSL** перед выполнением сценария. При нарушении возвращается ошибка `read_only_violation`.

Режим предназначен для безопасных запросов к данным без риска изменения.

## Ограничение записи в настройках пользователей

Регистр сведений **ИИА_НастройкиПользователей** содержит ресурс **ДоступнаЗапись**:

- **ДоступнаЗапись = Ложь** — запрещаются действия CreateReference, SetField, Write даже в режиме «Агент»
- **ДоступнаЗапись = Истина** (по умолчанию) — запись разрешена

Проверка выполняется в **ИИА_Сервер** при вызове DSL (ВыполнитьDSLСценарийССообщениями, ВыполнитьDSLСценарийБезСообщений) и в **ИИА_DSL** при определении флага `ТолькоЧтение`.

Настройка позволяет ограничить конкретных пользователей только чтением, не меняя тип диалога.

## Токенизация с обратным маппингом

Система использует продвинутую токенизацию вместо простой маскировки:

### Принцип работы:

1. **Загрузка данных из базы** — система загружает наименования контрагентов, ФИО сотрудников, адреса
2. **Токенизация** — реальные данные заменяются на токены `[КОНТР_1]`, `[ФИО_2]`, `[АДРЕС_3]`
3. **Сохранение маппинга** — соответствие токен↔значение сохраняется в сессии диалога
4. **Отправка в LLM** — LLM получает текст с токенами, не видя реальных данных
5. **Детокенизация** — в ответе LLM токены заменяются обратно на реальные данные

### Типы токенов:

| Токен | Описание | Пример |
|-------|----------|--------|
| `[КОНТР_N]` | Наименование контрагента | ООО "Ромашка" |
| `[ФИО_N]` | ФИО физлица/сотрудника | Иванов Иван Иванович |
| `[АДРЕС_N]` | Адрес | г. Москва, ул. Ленина, 15 |
| `[ИНН_N]` | ИНН | 7707083893 |
| `[КПП_N]` | КПП | 770701001 |
| `[ОГРН_N]` | ОГРН / ОГРНИП | 1027700132195 |
| `[ТЕЛ_N]` | Телефон | +7 (495) 123-45-67 |
| `[EMAIL_N]` | Email | info@company.ru |
| `[РС_N]` | Расчётный/корреспондентский счёт | 40702810000000000001 |
| `[БИК_N]` | БИК банка | 044525225 |
| `[СНИЛС_N]` | СНИЛС | 112-233-445 95 |

### Преимущества:
- LLM может логически работать с данными (сравнивать, группировать)
- Реальные данные не покидают периметр 1С
- Ответ LLM автоматически содержит корректные данные

### Реализация:

Логика токенизации находится в модуле **ИИА_Анонимизация**:

- `ТокенизироватьТекст(Текст, СсылкаДиалога)` — заменяет реальные данные токенами, сохраняет маппинг
- `ДетокенизироватьТекст(Текст, СсылкаДиалога, Маппинг)` — восстанавливает исходные данные из токенов
- `ЗагрузитьДанныеДляТокенизации()` — загружает справочные данные из базы (контрагенты, физлица, адреса)

Данные из базы кэшируются на время сеанса для повышения производительности. Справочники сортируются по длине наименования в убывающем порядке, что обеспечивает корректную замену при наличии вложенных строк.

## Общие принципы

- Агент не имеет привилегированных режимов — все операции выполняются от имени текущего пользователя с учётом RLS
- Валидация данных: агент не может выводить данные, которые не были успешно получены из базы (защита от галлюцинаций)
